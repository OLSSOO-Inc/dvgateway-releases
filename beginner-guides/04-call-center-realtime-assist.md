# 가이드 4 — 콜센터 실시간 상담원 어시스턴트

> "신입 상담원이 헷갈리는 질문을 받으면 옆에 선임이 붙어줘야 한다. 그런데 선임을 매번 부를 수도 없고…" AI가 상담원 옆에서 실시간으로 답변을 추천해 주는 시스템입니다.

## 🎯 이 가이드로 만들 수 있는 것

- **고객 발언 실시간 자막** — 상담원 화면에 고객 말이 텍스트로 표시
- **답변 추천** — AI가 회사 매뉴얼·FAQ를 검색해 답변 후보 제시
- **고객 감정 분석** — 화났는지·만족스러운지 화면 표시 (이탈 방지)
- **자동 통화 요약** — 통화 끝나면 1줄 요약 + 분류 태그 자동 생성
- **상담원 행동은 변하지 않음** — 평소 통화 그대로, AI는 옆에서만 보조

## 💼 이런 분들께 추천

- **콜센터 / 고객센터를 운영하는 기업**
- **신입 상담원 교육 기간이 길어 부담스러운 조직**
- **상담원 이직률이 높아 매번 교육이 반복되는 곳**
- **상담 품질 편차가 큰 조직** (베테랑 vs 신입)
- **상담 후처리(메모 작성·CRM 입력)에 시간이 많이 드는 조직**

## 💰 예상 비용 (상담원 1명 기준, 일 100통 처리)

| 항목 | 비용 |
|------|------|
| 실시간 음성 인식 (Deepgram) | 약 9,000원/월 |
| AI 답변 추천 (Claude / GPT) | 약 30,000원/월 |
| 자동 요약 | 약 5,000원/월 |
| **상담원 1명당 합계** | **약 44,000원/월** |

> 신입 교육 기간이 평균 1~3개월에서 1~2주로 단축된다는 사례 보고.
> 상담원 1인당 처리 통화량 20~30% 증가 효과.

## 🔑 가장 큰 차별점

**상담원은 평소처럼 전화 받기만 하면 됩니다.**

- AI가 통화를 가로채거나 끊지 않음 (= "monitor 모드"라는 비침투 방식)
- 상담원 화면에 보조 정보가 표시될 뿐, 고객은 AI의 존재를 모름
- 상담원이 AI 추천을 무시해도 통화는 정상 진행

→ 기존 상담원 워크플로 변경 최소, 도입 저항 적음.

## 📋 도입 절차

### 1단계 — 추천 답변 소스 준비 (2~5일)
AI가 추천할 답변의 "지식 베이스"를 준비합니다.

- 회사 FAQ 문서 (있는 그대로)
- 상담원 매뉴얼·매뉴얼 워드 파일
- 과거 우수 상담 사례 (있다면)
- 제품 카탈로그

→ PDF·워드·노션 페이지 등 그대로 업로드하면 AI가 자동 인덱싱.

### 2단계 — 상담원 화면 설계 (1주)
어떤 정보를 보여줄지 결정:
- 고객 발언 자막? (필수 권장)
- 답변 후보 3개 표시?
- 고객 감정 인디케이터 (😊 😐 😡)?
- 관련 매뉴얼 페이지 자동 표시?

> 너무 많은 정보는 오히려 산만 → 처음엔 자막 + 답변 후보만 시작 추천.

### 3단계 — 기존 콜센터 시스템 연동 (개발자 1~2주)
- 사내 CRM, 티켓 시스템, 상담원 데스크톱 앱과 연결
- 통화 종료 → 자동 요약을 CRM에 저장
- 통화 분류 태그(예: "결제 문의", "환불 요청") 자동 입력

### 4단계 — 파일럿 운영 (2주)
- 상담원 3~5명에게만 먼저 적용
- 일주일 단위로 답변 추천 정확도·만족도 점검
- 잘 안 되는 케이스는 1단계 지식 베이스 보완

### 5단계 — 전체 도입
- 파일럿 결과 좋으면 전 상담원에 확대
- 신입 상담원은 입사 첫날부터 사용

## 🌟 활용 사례

**중견 이커머스 콜센터** (가상, 상담원 30명)
> 도입 전: 신입 상담원 교육 3개월, 첫 달 처리량은 베테랑의 50%.
> 도입 후: 신입도 입사 2주차부터 베테랑의 80% 처리량.
> 베테랑 상담원은 어려운 케이스에 집중 → 전체 만족도 상승.
> 1년간 인건비 절감액 약 1억 8천만원, 시스템 비용 1,500만원.

**보험사 인입 상담** (가상)
> 보험금 청구 문의가 가장 까다로움 (약관 해석 필요).
> AI가 약관 PDF 1,000페이지를 인덱싱 → 고객 발언 중 즉시 관련 조항 표시.
> 상담원이 "잠시만요, 약관 찾아보겠습니다"하는 침묵 시간 80% 감소.
> 고객 만족도(NPS) 5점 상승.

**기술 지원 콜센터** (가상)
> 제품군이 50종 이상이라 신입은 어떤 매뉴얼을 봐야 할지 모름.
> 고객이 "△△ 모델인데…" 하는 순간 화면에 해당 매뉴얼 자동 표시.
> 평균 통화 시간 18분 → 12분으로 단축.

## 🔍 감정 분석은 어떻게 활용하나

화면 한쪽에 감정 그래프가 실시간 표시됩니다.

```
[고객 감정]
😊 ────────────┐
😐         ────┘
😡 ────────────
   0:00   2:00   4:00 (분)
```

- 감정이 급격히 부정적으로 바뀌면 상담원 화면에 알림
- 5분 이상 부정 감정 지속 시 슈퍼바이저에게 자동 알림 → 이탈 방지
- 통화 종료 후 부정 감정으로 끝난 통화만 자동 추출 → 콜백 우선 처리

## ⚖️ 관련 가이드와의 차이

| 가이드 | 누가 전화 받음? | AI 역할 |
|--------|----------------|---------|
| #1 AI 전화 상담원 | **AI가 직접 받음** | AI=상담원 본인 |
| #4 실시간 어시스턴트 (본 문서) | **사람이 받음** | AI=옆에서 보조 |

→ 두 방식을 **함께** 쓰는 게 가장 흔합니다. 1차 응답은 AI, 복잡한 건 사람에게 넘기되 그때부터 AI가 어시스턴트로 전환.

## ⚠️ 도입 시 주의점

- **상담원 동의 필수** — 노조가 있는 조직은 사전 협의
- **고객 안내** — "본 통화는 품질 향상을 위해 녹음·분석됩니다" 안내
- **AI 추천 맹신 금지** — 추천은 보조일 뿐, 최종 판단은 상담원
- **개인정보 분리** — 통화 분석 시 카드번호·주민번호 자동 마스킹
