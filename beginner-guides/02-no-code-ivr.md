# 가이드 2 — 코딩 없는 무인 안내 전화

> "AI까지 갈 필요는 없고, '1번 영업, 2번 기술지원' 같은 안내만 자동으로 되면 충분한데…" 그럴 때 가장 저렴한 선택입니다.

## 🎯 이 가이드로 만들 수 있는 것

- **영업시간 외 안내 멘트** — "지금은 영업시간이 아닙니다. 평일 9시-18시에…"
- **번호 메뉴(IVR)** — "1번 영업팀, 2번 기술지원, 3번 결제 문의"
- **간단 본인 인증** — "주민번호 뒷자리 7자리를 입력해 주세요"
- **공휴일·이벤트 안내** — 임시 공지 멘트 자동 전환
- **콜백 예약** — "1번 오전, 2번 오후를 선택해 주세요"

## 💼 이런 분들께 추천

- **모든 사업장** — 영업시간 안내는 어느 곳이나 필요
- **공공기관·도서관** — 부서별 메뉴 안내
- **AS 센터** — 제품군별 상담원 라우팅
- **택시·콜 서비스** — 콜 등록만 받기
- **소상공인** — AI 봇 도입 비용이 부담스러울 때 (대안으로)

## 💰 예상 비용

| 항목 | 비용 |
|------|------|
| DVGateway 라이선스 (Basic 이상) | 월 협의 |
| 클라우드 음성 인식 / 합성 | **0원** ⭐ |
| AI 대화 | **0원** ⭐ |
| **합계** | **사실상 라이선스 비용만** |

> 음성 인식·합성·AI를 **전혀 쓰지 않기 때문에** 외부 클라우드 비용이 0원입니다.
> 안내 멘트는 한 번만 녹음/합성해 파일로 저장해두고 무한 재사용.

## 🔥 왜 이렇게 저렴할까?

**"lite 모드"라는 특수 통화 프로파일을 사용합니다.**

- 일반 AI 봇: 통화 중 실시간으로 음성을 클라우드로 보내고 받음 → 통화 시간만큼 비용 발생
- lite 모드: 미리 녹음된 음성 파일을 재생 + 번호 입력만 받음 → 외부 API 호출 0건

같은 서버에서 일반 모드 대비 **3~5배 더 많은 동시통화**를 처리할 수 있어 라이선스 효율도 좋아집니다.

## 📋 도입 절차 (개발자 30분 작업)

### 1단계 — 멘트 시나리오 작성 (비전공자 작업, 30분~1시간)
워드 파일 또는 종이에 다음 형식으로 적습니다.

```
[메인 메뉴]
안녕하세요. OO기업입니다.
상담 분야를 선택해 주세요.
1번. 영업 문의
2번. 기술 지원
3번. 결제 문의
0번. 상담원 연결

[1번 선택 시]
영업팀으로 연결합니다. 잠시만 기다려 주세요.
→ 영업팀 내선 #100으로 전환

[2번 선택 시]
기술지원팀으로 연결합니다.
→ 기술지원 내선 #200으로 전환

[영업시간 외]
지금은 영업시간이 아닙니다.
영업시간은 평일 오전 9시부터 오후 6시까지입니다.
긴급한 문의는 010-1234-5678로 부탁드립니다.
```

### 2단계 — 멘트 녹음 또는 합성 (1시간)
세 가지 옵션 중 선택:

**옵션 A — 직접 녹음**
- 휴대폰으로 녹음 → 무료 변환 사이트(예: convertio.co)에서 WAV 16kHz로 변환
- 장점: 인간적인 느낌
- 단점: 멘트 수정 시 재녹음 필요

**옵션 B — AI 음성 합성 (한 번만)**
- ElevenLabs, Naver Clova Voice 등에서 텍스트 → 음성 변환 (1회 비용 약 1,000원)
- 장점: 멘트 수정 자유로움, 다양한 톤
- 단점: 약간의 합성 티가 날 수 있음

**옵션 C — 전문 성우 의뢰**
- 약 5~20만원으로 전문가 녹음
- 장점: 브랜드 인상에 가장 좋음
- 단점: 비용·시간

### 3단계 — 파일 업로드 (개발자 5분)
녹음된 파일을 게이트웨이 서버의 사운드 디렉토리에 복사하면 끝.

```bash
sudo cp 영업시간_안내.wav /var/lib/asterisk/sounds/ko/business-hours.wav
```

### 4단계 — 메뉴 흐름 설정 (개발자 30분, 한 번만)
간단한 분기 규칙을 코드 또는 설정으로 정의. (개발자에게 [기술 문서 17번](../sdk-guide/17-lite-mode-ivr.md) 전달)

### 5단계 — 전화번호 연결 → 테스트

## 🌟 활용 사례

**소형 서점** (가상)
> 평일 10시-19시 운영, 일요일 휴무.
> 영업시간 외 전화에 "주말은 휴무입니다. 인스타그램 @bookshop 으로 문의 부탁드립니다" 안내.
> 영업시간 중에는 "1번 신간 문의, 2번 재고 확인, 3번 사장님 직통"으로 분기.
> 도입 비용 월 0원(라이선스만), 직원 1명이 잡일에서 해방.

**중소 제조업체 AS 센터** (가상)
> 제품 모델별로 담당 엔지니어가 다름.
> "1번 A모델, 2번 B모델, 3번 C모델" 입력으로 자동 라우팅.
> 각 엔지니어 휴대폰까지 전달되도록 라우팅 → 응답률 95%로 상승.

## ⚖️ AI 봇 vs Lite 모드 비교

| 항목 | AI 봇 (가이드 #1) | Lite 모드 (이 가이드) |
|------|-------------------|----------------------|
| 자연어 대화 | ✅ 가능 | ❌ 불가 (번호 입력만) |
| 운영 비용 (월 1,000통) | 약 3만원 | 거의 0원 |
| 멘트 수정 | 시나리오 텍스트만 수정 | 음성 파일 재녹음 필요 |
| 도입 난이도 | 보통 | 쉬움 |
| 통화 처리량 (동시통화) | 보통 | 매우 많음 (3~5배) |
| 추천 통화 유형 | 복잡한 문의·예약 | 단순 안내·메뉴 라우팅 |

> **Tip**: 두 방식을 **함께** 쓰는 것도 가능합니다. 예) 평일 낮 영업시간엔 AI 봇, 야간엔 Lite 안내 멘트.
