# 가이드 7 — 코딩 없는 CRM·엑셀 자동 연동

> "전화 통화 후 매번 CRM에 수동 입력하는 게 일이다. 통화→자동 기록은 안 될까?" 코딩 없이 n8n/Zapier 같은 도구로 통화와 사내 시스템을 자동으로 이어줍니다.

## 🎯 이 가이드로 만들 수 있는 것

- **통화 발생 → CRM 자동 기록** — 발신자 번호·일시·녹음·요약이 자동 등록
- **신규 고객 자동 등록** — 처음 전화 건 번호는 자동으로 CRM에 새 카드 생성
- **통화 분류 자동 태깅** — "결제 문의", "환불", "기술 지원" 등 자동 분류
- **Slack/팀즈 실시간 알림** — 중요 키워드 감지 시 담당자에게 즉시 알림
- **엑셀·구글시트 자동 누적** — 일별 통화 통계 자동 정리
- **티켓 시스템 자동 등록** — Jira·Notion·Asana에 자동 이슈 생성
- **이메일 자동 발송** — 통화 끝나면 "오늘 상담 요약" 메일 발송

→ **모든 통합을 코드 한 줄 없이** 마우스 클릭만으로 설정.

## 💼 이런 분들께 추천

- **CRM을 쓰는 모든 영업 조직** — Salesforce·HubSpot·Pipedrive·국내 CRM
- **콜센터·고객 서비스팀** — 티켓 자동화로 응답 시간 단축
- **소규모 IT 인력 조직** — 개발자 없이도 자동화 구축 가능
- **마케팅팀** — 통화 데이터를 SNS 광고·이메일 캠페인과 연동
- **재무·회계팀** — 결제 관련 통화 자동 기록·증빙

## 💰 예상 비용

| 항목 | 비용 |
|------|------|
| n8n (자체 호스팅, 무료 오픈소스) | 0원 |
| n8n Cloud 또는 Zapier (관리형) | 월 2~5만원 (단계별) |
| 사내 자동화 설정 인력 시간 | 1회 1~2일 |
| **합계 (초기 1회 + 월 운영)** | **월 0~5만원** |

> 직원 1명이 매일 통화 후처리(CRM 입력·메모·요약)에 1시간 → 월 22시간 → 인건비 환산 약 100만원.
> 자동화 후 **월 95만원 절감**.

## 🧩 작동 방식 (그림으로 이해)

```
[전화 통화]
    ↓
[DVGateway] ─ 이벤트 발생 (call:new, call:ended, 회의록 등)
    ↓ Webhook (HTTP POST)
[n8n / Zapier] ─ 노드를 마우스로 연결
    ↓                ↓                ↓
[CRM 등록]    [Slack 알림]     [엑셀 추가]
```

n8n/Zapier는 "이런 일이 생기면 → 이걸 해라"를 시각적으로 연결하는 도구입니다.
DVGateway가 알려준 통화 정보를 받아서, 원하는 시스템에 자유롭게 전달.

## 📋 도입 절차

### 1단계 — 자동화 도구 선택 (10분)

| 도구 | 특징 | 추천 대상 |
|------|------|-----------|
| **n8n** (오픈소스) | 무료, 사내 설치, 한국어 가능, 제약 없음 | IT 인력 있는 조직 |
| **Zapier** | 가장 유명, 5,000+ 앱 연동, 비싸짐 | 빠르게 시작하고 싶은 소규모 |
| **Make** (구 Integromat) | n8n과 Zapier 중간, 가격 합리적 | 중간 규모 |
| **Flowise** | LLM 워크플로 특화 | AI 응답 자동화까지 원할 때 |

### 2단계 — DVGateway에서 Webhook 켜기 (5분)
관리 대시보드에서 다음 두 항목만 설정:
- 어떤 이벤트를 보낼지 (예: 통화 종료 시 / 회의록 완성 시)
- 어디로 보낼지 (n8n/Zapier에서 발급한 URL)

```
대시보드 → 설정 → Webhook
  ☑ 통화 종료 이벤트
  ☑ 회의록 생성 완료
  URL: https://n8n.example.com/webhook/abc123
  서명 키: ********  (보안용)
```

### 3단계 — n8n/Zapier에서 워크플로 만들기 (1~2시간, 한 번만)

**예시: 통화 종료 → CRM 등록 + Slack 알림**

```
[1. Webhook 수신]
  ↓ 통화 정보 도착
[2. CRM 검색] (전화번호로 기존 고객 조회)
  ↓
[3-a. 기존 고객] → CRM 업데이트 + 통화 기록 추가
[3-b. 신규 고객] → CRM에 새 카드 생성
  ↓
[4. Slack 알림]
  "📞 새 통화: {고객명} ({번호}) - {요약}"
```

각 단계는 마우스로 박스를 끌어다 놓고 연결만 하면 됩니다.

### 4단계 — 테스트 통화 → 확인 (10분)
- 실제 전화 한 통 → CRM 자동 등록·Slack 알림 확인
- 잘 되면 운영 시작

## 🌟 활용 사례

**중소형 부동산 중개소** (가상)
> 광고 보고 전화 건 고객 80%가 이름·연락처 미수집 상태로 통화 종료.
> n8n 자동화 도입:
> 1) 통화 시작 시 → 자동으로 CRM에 새 잠재 고객 카드 생성 (번호로)
> 2) AI 요약 → 카드의 "관심 매물" 칸에 자동 입력
> 3) 다음 날 영업사원 출근 시 → 자동으로 "어제 못 받은 콜 5건" 리스트 정리
> 결과: 잠재 고객 누락률 60% → 5%, 계약 성사율 35% 상승.

**B2B SaaS 회사** (가상)
> 무료 트라이얼 사용자가 영업팀에 전화하는 경우가 잦음.
> 자동화 도입:
> 1) 전화 수신 시 → 자동으로 Salesforce에서 해당 계정 조회
> 2) 트라이얼 사용 현황·이전 통화 기록을 상담원 화면에 자동 표시
> 3) 통화 종료 시 → Hubspot에 다음 액션 자동 생성 (제안서 발송 등)
> 영업사원이 통화 전 준비 시간 5분 → 0분.

**1인 운영 온라인 쇼핑몰** (가상)
> 사장님이 직접 모든 CS 통화 응대 + 수동 메모.
> n8n 자동화:
> 1) 통화 끝나면 → 자동으로 구글 시트에 "고객번호·일시·요약·다음 조치" 행 추가
> 2) "환불" 키워드 감지 시 → 텔레그램 봇으로 즉시 알림
> 3) 매주 일요일 → 한 주 통화 통계 메일 자동 발송
> 사장님은 통화만 하고 모든 기록은 자동.

**공공기관 민원실** (가상)
> 민원 전화를 매번 종이/엑셀에 수동 기록.
> 자동화:
> 1) 통화 종료 → AI 요약 + 분류 (시설·인허가·복지 등)
> 2) 자동으로 사내 민원 시스템에 티켓 생성
> 3) 담당 부서 슬랙 채널에 알림 + 처리 마감일 자동 설정
> 민원 누락 사고 0건, 처리 시간 평균 5일 → 2일.

## 🛠️ 자주 연결되는 시스템들

| 분류 | 예시 | n8n 노드 |
|------|------|---------|
| CRM | Salesforce, HubSpot, Pipedrive, Zoho, 국내 CRM | 모두 있음 |
| 협업 | Slack, Teams, Discord, 카카오워크 | 모두 있음 |
| 문서 | Google Docs/Sheets, Notion, Microsoft 365 | 모두 있음 |
| 티켓 | Jira, Asana, Linear, Trello, ClickUp | 모두 있음 |
| 이메일 | Gmail, Outlook, Mailgun, SendGrid | 모두 있음 |
| 채팅봇 | Telegram, Discord, Kakao 알림톡 | 모두 있음 |
| 데이터베이스 | PostgreSQL, MySQL, MongoDB, Airtable | 모두 있음 |
| 결제 | Stripe, 토스, 카카오페이 (API 있다면) | 대부분 있음 |

> n8n은 5,000+ 통합, Zapier는 7,000+ 통합을 기본 제공. 거의 모든 사내 시스템과 연결 가능.

## 🔐 보안 고려사항

- Webhook URL은 추측 불가능한 긴 토큰 포함 (자동 생성)
- 서명 키(HMAC-SHA256)로 위변조 방지
- 민감 정보(주민번호 등)는 자동 마스킹 후 전송 옵션 사용
- 통화 녹음 원본은 게이트웨이에 머무르고, n8n/CRM에는 텍스트만 전달 (선택)
- n8n 자체 호스팅 시 모든 데이터가 사내에 머무름

## 🆚 직접 코딩 vs n8n/Zapier

| 항목 | 직접 코딩 | n8n/Zapier |
|------|-----------|------------|
| 초기 구축 시간 | 1~4주 | 1~2일 |
| 수정 난이도 | 개발자 필요 | 마우스 클릭 |
| 외부 시스템 추가 | 그때마다 개발 | 노드 끌어 놓기 |
| 비용 (월) | 개발자 인건비 | 0~5만원 |
| 디버깅 | 로그 분석 | 비주얼 실행 추적 |
| 확장성 | 무제한 | 도구 한계 내 (대부분 충분) |

> **결론**: 80%의 자동화는 n8n/Zapier로 충분. 나머지 20%(매우 복잡한 비즈니스 로직)는 직접 개발.
