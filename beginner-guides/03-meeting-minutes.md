# 가이드 3 — 회의록 자동 생성

> "회의가 끝나면 누군가 회의록을 정리해야 하는데, 그 일이 매번 1~2시간씩 걸린다." 회의가 끝나는 순간 회의록이 이메일로 도착하게 만들 수 있습니다.

## 🎯 이 가이드로 만들 수 있는 것

- **회의 종료 → 1분 안에 텍스트 회의록 생성**
- **참여자별 발언 구분** — "김부장: …", "이대리: …"
- **자동 요약** — 전체 회의 요약 + 액션 아이템 추출 (AI 옵션)
- **검색 가능한 기록 보관** — 6개월 후에도 키워드 검색
- **법적 감사·컴플라이언스 대응** — 통화 녹음 의무가 있는 금융권에 유용

## 💼 이런 분들께 추천

- **경영진 회의가 잦은 조직** — 의사결정 기록 필수
- **연구개발팀** — 기술 회의 내용 정리
- **법무·컴플라이언스팀** — 외부 회의 증거 보전
- **다국적 협업 팀** — 영어/한국어 혼용 회의도 가능
- **컨설팅·법무 사무소** — 고객 미팅 기록
- **언론·방송** — 인터뷰 전사

## 💰 예상 비용

| 항목 | 비용 |
|------|------|
| 음성 인식 (Deepgram 다중 화자) | 회의 1시간당 약 300원 |
| 요약 (Claude / GPT, 선택) | 회의 1시간당 약 200원 |
| **합계** | **회의 1시간당 약 500원** |

> 직원 1명이 1시간 회의록 정리에 1시간 → 시급 2~5만원으로 환산하면 회의당 **2.5~5만원 절약**.
> 매주 5개 회의를 한다면 월 **50~100만원** 인건비 절감 효과.

## 📋 도입 절차

### 1단계 — 회의 방식 결정 (10분)

**옵션 A — 일반 전화 회의** (가장 흔함)
- 사내 PBX의 회의실 번호로 전화 걸기 → 회의 진행 → 끊으면 자동 회의록 생성
- 장점: 별도 앱 설치 불필요, 누구나 휴대폰으로 참여 가능
- 추천 대상: 외부 거래처 포함 회의

**옵션 B — VoIP 소프트폰 사용**
- 회사 노트북/PC에서 마이크로 참여
- 장점: 음질이 가장 좋음
- 추천 대상: 사내 회의 위주

**옵션 C — 휴대폰 SIP 앱** (Zoiper, Linphone 등)
- 휴대폰을 사무실 내선처럼 사용
- 추천 대상: 재택·외근이 많은 조직

### 2단계 — 시스템 설정 (개발자 30분, 한 번만)
- 회의실 내선번호 정의 (예: 5500, 5501, 5502…)
- 회의 종료 시 자동 회의록 생성 활성화
- 회의록 저장 위치 지정 (이메일 발송 / 사내 위키 자동 게시 등)

### 3단계 — 시범 회의 1회 (30분)
- 3-4명이 실제로 회의 진행
- 회의 종료 직후 회의록 확인 → 화자 구분, 정확도 점검

### 4단계 — 운영 정착
- 첫 1~2주는 자동 회의록과 사람이 정리한 회의록 병행
- 문제 없으면 자동 회의록으로 전환

## 📄 생성되는 회의록 형식

**JSON (시스템 연동용)**
```json
{
  "conferenceId": "5500",
  "startTime": "2026-05-20T14:00:00",
  "endTime": "2026-05-20T15:23:00",
  "participants": ["김부장", "이대리", "박과장"],
  "transcript": [
    { "speaker": "김부장", "time": "14:00:15", "text": "오늘 안건은…" },
    { "speaker": "이대리", "time": "14:01:30", "text": "지난주 진척은…" }
  ],
  "summary": "다음 분기 마케팅 예산을 30% 증액하는 방향으로…",
  "actionItems": [
    "이대리: 다음 주까지 예산안 작성",
    "박과장: 광고 대행사 3곳 견적 수집"
  ]
}
```

**TXT (사람이 읽기 좋음)**
```
========================================
회의 일시: 2026년 5월 20일 14:00 - 15:23
참석자: 김부장, 이대리, 박과장
========================================

[14:00:15] 김부장: 오늘 안건은 다음 분기 마케팅…
[14:01:30] 이대리: 지난주 진척은…
...

[요약]
다음 분기 마케팅 예산을 30% 증액하는 방향으로 논의.
경쟁사 분석 결과 SNS 광고 비중을 늘리기로 결정.

[액션 아이템]
- 이대리: 다음 주까지 예산안 작성
- 박과장: 광고 대행사 3곳 견적 수집
```

## 🌟 활용 사례

**스타트업 임원진 미팅** (가상)
> 매주 월요일 임원진 회의 1.5시간.
> 도입 전: 한 명이 노트북으로 받아 적고 그 주에 정리 (총 2시간).
> 도입 후: 회의 끝나면 5분 안에 슬랙으로 회의록·요약·할 일이 도착.
> 임원진은 통화만 하면 되고, 비서는 그 시간에 다른 업무.

**법무법인 외부 상담** (가상)
> 변호사가 의뢰인과 전화 상담 시 자동 녹취 + 텍스트화.
> 상담 직후 의뢰인에게 "오늘 상담 요약본"을 메일로 발송 → 신뢰도 상승.
> 분쟁 발생 시 정확한 기록 제공.

**기술 컨설팅 회사** (가상)
> 매번 다른 고객사와 기술 미팅.
> 회의록을 사내 검색 가능한 위키에 자동 게시 → 6개월 후 비슷한 문의가 와도 과거 답변을 즉시 찾음.

## 🔒 보안·법적 고려사항

- **녹음 동의**: 한국 법상 통화 녹음은 당사자 1인 동의로 가능하지만, 외부 회의에서는 시작 시 알리는 것이 권장
- **개인정보 보호**: 회의록에 주민번호·계좌번호가 포함될 수 있음 → 자동 마스킹 옵션 활성화 가능
- **저장 위치**: 게이트웨이 서버 자체 저장 또는 사내 NAS — **외부 클라우드 미전송 가능** (음성 인식만 외부 API)
- **온프레미스 STT**: 외부에 음성 데이터조차 보내고 싶지 않다면 사내 GPU 서버에 자체 STT 운영 가능 (별도 도입 비용 발생)

→ 금융·의료·법무 등 규제 산업은 [한국 AI SaaS 규제 가이드](../korea-ai-saas-regulatory-guide.md) 참조.

## 📊 정확도 (현실적 기대치)

| 회의 환경 | 인식 정확도 | 화자 구분 |
|-----------|------------|-----------|
| 조용한 사무실, 표준어 | 95% 이상 | 95% 이상 |
| 사무실, 일부 사투리 | 90% | 90% |
| 카페·노이즈 있음 | 85% | 80% |
| 여러 명 동시 발언 | 변동 | 70% |
| 영어/한국어 혼용 | 90% (다국어 모델 사용 시) | 85% |

> 100% 자동화는 불가능하지만, 90%만 자동화돼도 사람이 보정하는 시간이 80% 줄어듭니다.
