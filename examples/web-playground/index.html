1. 테넌트 자격증명

2. 데모 템플릿

아래에서 체험할 기능을 골라요. 선택하면 오른쪽에 안내가 떠요.

    3. 발신

    먼저 Connect 하세요. 연결 후 게이트웨이에 등록된 발신표시번호·과금번호를 자동으로 불러옵니다.

    상대방이 전화를 받은 뒤 어디로 연결할지 지정하는 번호예요. 보통 자동응답(IVR) 진입 번호나 상담사 내선을 넣어요.

    고정값 (테넌트 등록값 · 변경 불가) 🔒
    발신표시번호—
    과금번호—

    변경하려면 관리자에게 문의하세요.

    4. 프로바이더 API 키

    데모용 STT/TTS provider 키. 비워두면 게이트웨이의 기본 설정을 사용합니다.

    TTS

    목소리는 한국어 기본 음성으로 자동 선택돼요. 다른 음성이 필요하면 SDK를 직접 사용해 주세요.

    STT (Mode A 전용)

    STT 결과는 게이트웨이가 callinfo로 다시 전달하므로 브라우저 직접 호출은 지원하지 않습니다.

    5. 활성 통화

    아직 활성 통화가 없습니다.

    템플릿을 선택하세요

    라이브 이벤트 0 events

    현재 템플릿의 핵심 코드 — 그대로 복사해 자기 앱에 붙여넣을 수 있습니다. SDK 가이드 docs/sdk-guide/14-tenant-fixed-audio-guide.md 의 흐름과 동일합니다.

    // 템플릿을 선택하면 여기에 표시됩니다.
    0 events

    👋 DVGateway 체험 페이지에 오신 것을 환영해요

    여기는 전화 통화에 음성을 보내거나, 통화에서 들리는 음성을 처리하는 여러 기능을 직접 눌러보며 체험해 볼 수 있는 페이지예요.

    🔑
    STT·TTS·AI 데모는 프로바이더 API 키가 있어야 동작해요. 왼쪽 「4. 프로바이더 API 키」에서 사용할 STT(예: Deepgram) · TTS(예: ElevenLabs) 키를 먼저 저장하세요. 키를 넣지 않으면 음성 합성·전사 데모는 오류가 납니다.
    1. 왼쪽에서 서버 정보를 넣고 연결해요. (관리자에게 받은 주소·테넌트·비밀번호)
    2. 「4. 프로바이더 API 키」에 STT·TTS 키를 저장해요. (Callinfo 모니터만 볼 거면 건너뛰어도 돼요)
    3. 테스트용 전화를 한 통 걸어요. 게이트웨이가 통화를 인식하면 진행 중인 통화 목록에 나타나요.
    4. 왼쪽 데모 목록에서 마음에 드는 항목을 골라 클릭해요.
    5. 오른쪽 화면에 안내된 대로 버튼을 눌러 보세요. 결과가 통화에 즉시 반영돼요.
    💡
    처음이라면 Callinfo 모니터 데모부터 시작해 보세요. API 키 없이도 동작하고, 전화 이벤트가 실시간으로 나타나는 모습만 봐도 게이트웨이가 어떻게 동작하는지 감이 잡혀요.
